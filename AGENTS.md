## Структура проекта и конвенции

**Точка входа:**
*   Приложение — это стандартный веб-сервер на Go. Основная точка входа — `main.go:main()`.
*   Запуск приложения: `go run main.go`. Сервер прослушивает `http://localhost:8080`.

**Ключевая конечная точка API:**
*   **Эндпоинт:** `/calculate`
*   **Метод:** `POST`
*   **Обработка:** Обрабатывается `handlers.go:calculateHandler`.
*   **Тело запроса:** Требует полезную нагрузку JSON, соответствующую структуре `models.CalcRequest`.

**Особенности бизнес-логики (Обязательно знать):**
*   `calculateHandler` не принимает тип операции в теле запроса. Вместо этого он случайно выбирает одну из четырех поддерживаемых операций ("сложение", "вычитание", "умножение", "деление") с помощью `rand.Intn(len(actions))` внутри `handlers.go:calculateHandler`.
*   **Операции:**
    *   `Add()`: `A + B`
    *   `Subtract()`: `A - B`
    *   `Multiply()`: `A * B`
    *   `Divide()`: `A / B` (Проверяет деление на ноль в `models.go:CalcRequest.Divide()`).

**Рабочий процесс тестирования:**
*   Тесты комплексные и используют стандартные практики тестирования Go, что видно в `models_test.go`.
*   Запуск всех юнит-тестов: `go test ./...`

## Критические уязвимости/Замечания
*   **Взаимодействие с API:** Автоматические вызовы конечной точки `/calculate` должны учитывать случайный выбор операции, присущий `handlers.go`.
*   **Тестирование:** При написании тестов убедитесь, что все методы `CalcRequest` покрыты, уделяя особое внимание крайним случаям (например, отрицательные числа, деление на ноль), как показано в `models_test.go`.
